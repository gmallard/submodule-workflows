Introduction

This page describes the salted supermodule example. The intent of this example is to show how
to set up a super module and two submodules, and then work within that environment.

System Requirements

Required components and environment are:

  • An installed and working version of git
  • Successful administrative setup as described in Administrative Setup

Instructions

Using your own user ID, change to the salted-super subdirectory, and run the first setup script:

  1. ./gen.shared.sh

Use multiple terminals, log in as yourself, or one of the git users created during setup, and:

  1. Create a private work directory
  2. CD in to that directory
  3. Place the project scripts in your $PATH

Using each user instance, clone the environment once by using:

  1. clone-001.sh

Follow that by running use case 1:

  1. uc-001.sh

The latter script may be executed multiple times from all user instances (in any order).
