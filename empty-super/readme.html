The Empty Supermodule Examples


Introduction

This page describes the empty supermodule examples.

System Requirements

Required components and environment are:

  • An installed and working version of git
  • Successful administrative setup as described in Administrative Setup

Instructions

Using your own user ID, change to the empty-super subdirectory, and run the scripts in the following order:

  1. ./gen.shared.submods.sh
  2. ./gen.shared.empty.super.sh
  3. ./set-perms.sh
  4. ./add.submod.sh

In a separate terminal (or terminals), change to one of the supermodule directories created by ./add.submod.sh. Then:

  1. Add the empty-super directory to $PATH
  2. uc-001.sh

The latter script may be executed multiple times from either of the repo direcotries created by ./add.submod.sh.

Other users in the git group may participate by:

  1. Change to directory $HOME
  2. Add the empty-super directory to $PATH
  3. get.submods.sh

And then:

  1. uc-001.sh

The latter script may then be executed multiple times from any git user.
