Introduction

This page describes the empty supermodule example. The intent is to show how to
begin with a bare public super repository that has not been set up with submodules.
Then to subsequently work with that environment.

System Requirements

Required components and environment are:

  • An installed and working version of git
  • Successful administrative setup as described in Administrative Setup

Instructions

Using your own user ID, change to the empty-super subdirectory, and run these scripts in the following order:

  1. ./gen.shared.submods.sh
  2. ./gen.shared.empty.super.sh
  3. ./add.submod.sh

In a separate terminal (or terminals):

  1. change to the supermodule directory created by ./add.submod.sh
  2. Add the empty-super directory to $PATH
  3. uc-001.sh

The latter script may be executed multiple times from the repo directory created by ./add.submod.sh.

Other users in the git group may participate by:

  1. Change to directory $HOME
  2. Add the empty-super directory to $PATH
  3. get.submods.sh
  4. cd super

And then:

  1. uc-001.sh

The latter script may then be executed multiple times by any git user.
