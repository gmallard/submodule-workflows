Background

Before attempting to run either the salted supermodule or the empty supermodule example you must setup the required execution environment. The tasks to be accomplished are:

  • Define the user ID that will be used for git administration
  • Define a group to be used for repository sharing
  • Define two user IDs to be used for the tests
  • Attach all required user IDs to the sharing group
  • Create a common directory where public repositories will reside
  • Set the common directory sharing permissions properly

Most of these tasks will require root authority, or at the very least the ability to sudo the required commands.

Sharing Philosophy

These examples are crafted to run in an environment where the default umask value is 0002.

This is the normal case for Redhat style Linux systems. These examples will also run in other Linux environments (e.g. Debian) where that is not the case. Code in the examples enforces this behavior.

Requirements for:

  • Other *nix systems
  • Windows systems
are unknown. Good luck!

Task List

Note that you can accomplish these tasks using any of the standard UIs available. I show the command line versions here, and assume that you are comfortable with basic system administration.

In all cases, you may need to consult your system man pages.

Define the git Administrative User

You should be prompted for a password, and some other administrative details. 

adduser gitadmin

Define the git Sharing Group

addgroup git

Define Test Users

adduser gua
adduser gub

Attach Users to the Sharing Group

usermod -a -G git gitadmin
usermod -a -G git gua
usermod -a -G git gub
usermod -a -G git $USER

Create the Public Repository Directory

The repository directory may be created at any convenient location. 

mkdir -p /home/opt/public/repos

Set the Public Repository Directory Permissions

chown gitadmin.git /home/opt/public/repos
chmod 2775 /home/opt/public/repos
