Merge Conflict Resolution

git push Fails

If you attempt to push, e.g.: 

git push

and the result is like:

 ! [rejected]        master -> master (non-fast forward)
error: failed to push some refs to '/home/gallard/subtut/public/super/.git'

You probably need to pull, and (assuming a successful merge), then push again. But note: this pull could yield a merge conflict

git pull
.
.
git push

Normal Merge Conflicts

If you get merge conflicts when you 'pull and push', and they are normal file conflicts, fix them per normal procedures. This will look something like: 

git pull
remote: Counting objects: 5, done.
remote: Compressing objects: 100% (2/2)remote: , done.
remote: Total 3 (delta 0), reused 0 (delta 0)
Unpacking objects: 100% (3/3), done.
From /home/gallard/subtut/public/a/
   87ff7c1..7eb223f  master     -> origin/master
Auto-merged a.txt
CONFLICT (content): Merge conflict in a.txt
Automatic merge failed; fix conflicts and then commit the result.

Fix per normal git procedures: 

vi a.txt
git add a.txt
git commit

Merge Conflicts in a Supermodule

If you are in a super-module, and get conflict error messages during a 'pull and push' like: 

git pull
remote: Counting objects: 4, done.
remote: Compressing objects: 100% (3/3), done.
remote: Total 3 (delta 2), reused 0 (delta 0)
Unpacking objects: 100% (3/3), done.
From /home/gallard/subtut/public/super/
   7ca8903..1440a56  master     -> origin/master
Auto-merged a
CONFLICT (submodule): Merge conflict in a - needs 7d60be49cea14ce00dba02db768850a959e45d6a

you should run a: 

git submodule update
git add a
git commit -m "Update submod for merge conflict"
#  And then remember the push
git push
